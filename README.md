# 01-GitHub-Foundation

## Overview

This repository documents the professional GitHub foundation for my IT career roadmap. It defines how my GitHub profile, repositories, lab documentation, screenshots, troubleshooting notes, and project evidence will be organized.

The purpose of this repository is to create a consistent documentation standard for every technical project I build moving forward.

## Standard Repository Structure

Each major technical repository should include:

```text
README.md
docs/
screenshots/
assets/
scripts/
notes/
```

### Folder Purpose

| Folder         | Purpose                                                    |
| -------------- | ---------------------------------------------------------- |
| `docs/`        | Written documentation, guides, and explanations            |
| `screenshots/` | Lab screenshots and proof of completion                    |
| `assets/`      | Diagrams, images, and supporting files                     |
| `scripts/`     | PowerShell, Bash, Python, or automation scripts            |
| `notes/`       | Study notes, troubleshooting notes, and command references |

## Screenshot Standards

Screenshots should be used as proof of completed work.

Each screenshot should include:

* Clear file name
* Short description
* Date when useful
* Related lab step
* No sensitive personal information
* No exposed passwords, tokens, or private keys

Example screenshot names:

```text
01-proxmox-dashboard.png
02-windows-server-static-ip.png
03-dns-manager-zone-created.png
04-active-directory-users-ou.png
05-dhcp-scope-created.png
```
